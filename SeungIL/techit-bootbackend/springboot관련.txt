gradle - 버전설정/라이브러리 땡겨오기

mavenCentral() - 라이브러리를

정적 컨텐츠 : 파일 그대로를 웹브라우저(고객)한테 전달해주는 것
MVC(model, view, controller)와 템플릿 엔진 : 서버에서 좀 변형해서(html을 좀 바꿔서) 전달해주는것
API : 아이폰/ 안드로이드 개발한다면 서버에서는 요즘 제이슨이란 데이터 구조 포멧으로 클라이언트한테 데이터를 전달하는 방식

스프링 부트는 정적컨텐츠를 자동으로 제공을 함

view는 그냥 화면을 그리는데 모든 역량을 집중해야됨
controller나 model과 관련된 부분은 비즈니스 로직과 관련이있거나 내부적인걸 처리하는데 집중을 해야됨
그래서 쪼갠거임

타임리프 템플릿에 장점은 서버없이 소스만으로 열어봐도 뭔지 알 수 있음. 서버없이 그냥 볼때 편하게 볼 수 있음
실제로 서버를 타서 볼때는 내용이 다르게 보임

@ResponseBody -> html의 body부분에 데이터를 직접 넣겠다

xml 방식 <HTML></HTML>이게 xml 방식
