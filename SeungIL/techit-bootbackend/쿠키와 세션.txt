서버 | -> 로그인 성공 | 클라이언트

쿠키 : 서버에서 클라이언트로 보내져서 브라우저에 저장되는 아주작은 크기의 데이터
키/값 구조, 유효기간 있음(지나면 삭제)

http 통신에서는 각각의 요청과 응답이 독립적, 따라서 이전의 작업 기억 못함 따라서 이걸 어딘가 남겨놔야 함

http에서 각각의 응답과 요청이 종료됐을때 저장이 안되는 특성을 갖고 있음 이걸 Stateless라고 함.

우리가 사용하는 서비스 수천 수만 수백만명이 동시에 연결돼 있기 때문에 Stateless특성이 없으면 엄청난 자원소모가 됨.

서버가 클라이언트한테 쿠키를 보내주면 클라이언트는 로그인을 하고 다른 페이지로 이동해도 로그인이 풀리지 않음.

아주 원초적인 http환경에선 당연하지 않음 

쿠키에는 민감한 데이터는 최대한 안담기게 해야함 -- 공격, 등에 취약

세션 

쿠키의 단점을 보완하기 위해 나온 개념

크라이언트에 상태를 나타내는 쿠키는 서버측에 세션저장소에 담김

클라이언트에서 굳이 불필요하게 민감정보등을 갖지 않음
