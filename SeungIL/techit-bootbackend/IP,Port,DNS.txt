IP, Port, DNS

-네트워크 
2대 이상의 컴퓨터가 연결된 통신망(상호작용)
어떻게 데이터가 오가는지 이해하는 것이 네트워크의 핵심

host - 다른 컴퓨터

연결을 위한 매개체 -> 스위치(장비) : 동일한 네트워크에서 호스트 간의 통신을 연결해주는 매개체 
스위치는 다른 네트워크엔 접속 불가함. 즉 인터넷 환경에 접근 어려움 따라서 라우터를 씀 얜 다른 네트워크를 접근할 수 있음 라우터는 공유기 

거대 네트워크 == 인터넷

-IP 주소
IP( Internet Protocol) : 컴퓨터 간 데이터를 주고받는 네트워크 계층의 규약 , 데이터 전달에 필요한 (목적지 컴퓨터 정보가 필요)

IPv4 / IPv6 
32비트 / 128비트

아직 IPv6가 완전히 자리잡진 않고 병행으로 사용 됨.

11000000101010000000000000000011
실제 값은 2진수로 구성. 우리가 실제로 사용할 땐 10진수
 32비트 주소를 8비트 씩 분할
11000000|10101000|00000000|00000011
한 파트가 1옥텟 
위의 2진수를 10진수로 변환 하면
192.168.0.3

0.0.0.0 ~ 255.255.255.255

공인 IP vs 사설 IP

ISP(Internet Service Provider) - 일반적인 가정집
ISP는 공인IP를 부여함

공인 IP(Public IP) : 전체 인터넷 망에서 고유하게 식별 가능한 주소. IPv4 체계에서 자원 부족

컴퓨터는 공인IP 대신 사설 IP를 할당받아 사용 

공유기에 공인 IP를 하나 할당하고 새로운 네트워크에 IP를 할당함. 이러면 공인 IP를 굳이 모두 할당하지 않아도 IPv4에서 관리가능
즉 하나의 공인 IP에서 수많은 사설 IP 할당이 가능.

사설 IP(Private IP) : 가정의 LAN과 같은 네트워크에서 할당되는 주소 , 컴퓨터에서 조회되는 IP

공인 IP는 최대한 노출 안하는게 좋음

통신사에서 공인 IP를 할당(ISP) -> 공유기(라우터) -> 사설 IP 할당

127.0.0.1 (특수한 IP) 컴퓨터 관점에서 자기자신을 가르키기위해 약속 된 주소 (localhost)
Loopback Network Interface

-Port 
하나의 컴퓨터에는 다양한 서비스가 실행중
port는 서비스를 구분하는 역할 

HTTP 80 - 보안 o
HTTPS 443 - 보안 x
SMTP 25 - 메일 전송
FTP 21 - 파일 전송

접근하려는 서비스의 목적지 포트를 정확하게 설정

-DNS

구글에 접속하려면 www.google.com 입력이 정석 방법

DNS (Domain Name Server) : URL을 해석하여 IP주소로 반환하는 서버

통상 사용하는 URL을 IP주소로 바꾸는 중간다리 역할을 하는 DNS server가 있음

DNS server는 국가, 기업등이 운영 전세계 DNS는 연결되어 있음
장애 발생시 막대한 피해로 이어짐
